# 软件工程作业

## 一、 第一章

1.   什么是软件危机？为什么会产生软件危机？

     1.   软件危机是指在软件开发和维护中所产生的一系列严重的问题，一是如何开发软件，满足用户对软件的需求，二是如何维护数量众多的已有软件。
     2.   软件危机产生的原因
          1.   软件开发无计划性
          2.   软件需求不充分
          3.   软件开发过程无规范
          4.   软件产品无评测手段

2.   何谓面向对象软件工程？

     它以面向对象程序设计为基础，其开发过程可包含以下阶段：

     OO分析与对象抽取 -> 对象详细设计 -> 面向对象的编码与测试

3.   什么是形式化软件开发方法？

     形式化软件开发方法就是软件工程的高级研究课题之一。它是一种基于数学的开发技术，主要采用数学的方法来描述系统的性质，例如程序变换和程序验证等。

## 二、第二章

1.   什么是软件生存周期？

     一个软件从开始立项起，到废弃不用止，统称为软件的生命周期。

2.   传统的瀑布模型把生存周期划分为哪些阶段？

     一般可划分为以下阶段：

     需求定义与分析 -> 总体设计 -> 详细设计 -> 编码 -> 测试 -> 使用维护

3.   什么是快速原型法？

     快速原型模型的中心思想是：先建立一个能够反映用户主要需求的原型，让用户实际看一下未来系统的概貌，以便判断哪些功能是符合需要的，哪些方面还需要改进。然后将原型反复改进，直至建立完全符合用户要求的新系统。

## 三、第三章

1.   需求分析的任务是什么？

     需求分析主要有两个任务：第一是通过对问题及其环境的理解、分析和综合建立分析模型；第二是在完全弄清用户对软件系统的确切要求的基础上，用“软件需求规格说明书”把用户的需求表达出来。需求分析的任务就是为了明确要开发的是一个什么样的系统，而不是去怎么实现这个系统。

2.   需求分析需要经过哪些步骤？

     需求分析、需求提炼、需求描述、需求验证。

3.   什么是结构化分析？

     结构化分析是使用DFD、DD、结构化语言、判定表和判定树等工具，来建立一种新的、称为结构化说明书的目标文档。

4.   需求规格说明书由哪些部分组成？

     引言、信息描述、功能描述、行为描述、质量描述、接口描述、其他描述。

## 四、第四章

1.   面向对象有哪些基本特征？

     抽象、封装、继承和多态，构成了OO的基本特征。

2.   UML提供了哪9种图？

     5种静态图：用例图、类图、对象图、构件图和部署图

     4种动态图：状态图、时序图、协作图和活动图

## 五、第五章

1.   什么是软件需求？可以从哪些方面描述软件需求？

     -   软件需求是指用户对软件系统在功能、行为、性能、设计约束等方面的期望和要求。
     -   可以从功能需求、非功能需求（如性能需求、安全需求、可用性需求等），以及设计约束（如技术选型、开发环境、运行环境等）等方面来描述软件需求。

2.   有哪两种主要的需求模型？

     功能模型和数据模型

3.   基于用例的面向对象的需求建模包括哪些步骤？

     识别参与者、识别用例、定义用例规格说明书等步骤。
